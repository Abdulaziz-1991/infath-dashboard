# لوحة إنفاذ — الحل الكامل (٦ منصات) بتحديث فوري، بالنقر فقط

هذا الحل يشمل **جميع المنصات الست** بدون استثناء: وصلت، مباشر، السعودية
للمزادات (عبر API مباشر)، والدال، سومتك، دار المزادات (عبر متصفح Chrome
حقيقي مضمّن داخل صورة Docker). ما فيه أي تعطيل أو تقليص — هذا التشغيل
الكامل.

## الملفات المطلوبة (٦ ملفات، كلها جاهزة عندك الآن)
- `app.py`
- `infath_auto_updater.py`
- `infath_live_dashboard.html`
- `infath_data.json`
- `requirements.txt`
- `Dockerfile`

---

## الخطوة 1: حساب GitHub
1. روح لـ **github.com** → **Sign up** (تخطّى لو عندك حساب)

## الخطوة 2: مستودع جديد
1. اضغط **New** (أو ‎+‎ → **New repository**)
2. اسم مثل `infath-dashboard`، اخترها **Public**
3. **Create repository**

## الخطوة 3: ارفع الملفات
1. **Add file** → **Upload files**
2. اسحب وأفلت الملفات الستة: `app.py`, `infath_auto_updater.py`,
   `infath_live_dashboard.html`, `infath_data.json`, `requirements.txt`,
   `Dockerfile`
3. **Commit changes**

## الخطوة 4: حساب Render
1. روح لـ **render.com** → **Get Started** → **Sign up with GitHub**
2. وافق على صلاحية القراءة للمستودع

## الخطوة 5: أنشئ Web Service
1. **New +** → **Web Service**
2. اختر مستودع `infath-dashboard`
3. Render سيكتشف وجود `Dockerfile` تلقائياً ويختار بيئة **Docker** بنفسه —
   **لن تحتاج تعبئة Build Command أو Start Command إطلاقاً**، الحقول هذي
   تختفي تلقائياً لأن الـ Dockerfile يحدد كل شيء
4. عبّي فقط:

| الحقل | القيمة |
|---|---|
| **Name** | أي اسم، مثل `infath-dashboard` |
| **Region** | Frankfurt |
| **Instance Type** | Free (أو Starter لو تبي أداء أفضل — اختياري) |

5. **Create Web Service**

## الخطوة 6: انتظر
بناء صورة Docker (تثبيت Chromium) يأخذ وقت أطول من حل المنصات الثلاث —
عادة **٤ إلى ٦ دقائق** أول مرة فقط. راقب السجل حتى تشوف
`Your service is live 🎉`.

## الخطوة 7: افتح لوحتك
رابط جاهز بأعلى الصفحة، شكله:
```
https://infath-dashboard.onrender.com
```
احفظه. اللوحة الآن تعرض **المنصات الست كلها** وتتحدث تلقائياً كل ٢٠ دقيقة.

---

## ملاحظات تستاهل انتباهك (لا تؤثر على اكتمال الحل، لكن مهم تعرفها)

**1) أول دورة تحديث بعد التشغيل**
أول مرة يبدأ فيها التطبيق، يحتاج دقيقة أو دقيقتين ليجمع بيانات المنصات
الست (الثلاث التي تستخدم Chrome أبطأ من الثلاث ذات الـ API). خلال هذي
الدقائق القليلة قد تشوف اللوحة بالبيانات التجريبية المرفقة، ثم تتحدّث
تلقائياً بمجرد اكتمال أول دورة حقيقية — بدون أي تدخل منك.

**2) النوم بعد عدم الاستخدام (الباقة المجانية)**
الباقة Free من Render تنام بعد ~١٥ دقيقة بدون زيارات، وأول فتح بعدها يأخذ
حوالي ٣٠-٤٥ ثانية. **الحل المجاني:** سجّل بـ **uptimerobot.com** وأضف
رابط لوحتك ليُزار كل ٥ دقائق تلقائياً — يبقيها مستيقظة ويبقي التحديث
التلقائي شغّالاً باستمرار، بالنقر فقط وبدون أي تكلفة.
*(أو ترقية لباقة Starter ‎(~7$/شهر)‎ لإلغاء النوم نهائياً.)*

**3) استهلاك الذاكرة**
الباقة المجانية توفر 512MB رام. تشغيل Chrome (واحد في كل مرة بالتتابع،
وليس بالتوازي) يستهلك ذاكرة بشكل ملحوظ أثناء كل دورة تحديث. إذا لاحظت
تعثر أو إعادة تشغيل التطبيق من Render أثناء دورة التحديث، الحل هو
الترقية لباقة Starter (رام أكبر) — هذا قيد عام في أي باقة مجانية وليس
خطأ بالكود.
